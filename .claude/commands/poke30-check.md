# poke30-check — サイトの動作確認

公開前に、サイトが壊れていないかを確認するスキル。

## 手順

1. **型チェック**
   ```bash
   cd "/Users/yugamotoyama/Downloads/Dラボ　/ポケモンカード" && npm run check
   ```
   - 型エラーがあれば修正。

2. **本番ビルド ＝ データ検証**
   ```bash
   cd "/Users/yugamotoyama/Downloads/Dラボ　/ポケモンカード" && npm run build
   ```
   - `src/data/validate.ts` がビルド時に走り、以下を自動チェックする:
     - id の重複 / 欠落、name・url・source の欠落
     - 日付形式、applyStart > applyEnd の矛盾
     - **応募期間が 2026-06-06〜2026-10-31 から外れていないか**
     - 実店舗/チェーンの都道府県名が47都道府県と一致しているか・city があるか
   - 失敗したらエラー文に該当 id が出るので、stores.ts を直して再実行。

3. **リンク生存チェック（最重要）**
   ```bash
   cd "/Users/yugamotoyama/Downloads/Dラボ　/ポケモンカード" && npm run check:links
   ```
   - 各店の url に実際にアクセスして応答を確認する。
     - ✅ 200番台＝生きている／🔒 403等＝ボット拒否（人は開ける・URLは有効）／❌ 404・5xx・接続不可＝リンク切れ。
   - **受付中(open)の店が ❌ なら exit 1 で失敗**＝公開してはいけない。直してから再実行。
   - 受付予定(scheduled)の ❌ は要見直し（致命的ではない）。

4. **生成物の確認**
   ```bash
   cd "/Users/yugamotoyama/Downloads/Dラボ　/ポケモンカード" && ls dist && ls dist/online
   ```
   - トップ・各都道府県・online・404 のHTMLが生成されているか確認。

4. **プレビュー描画チェック**（任意・見た目を確認したいとき）
   ```bash
   cd "/Users/yugamotoyama/Downloads/Dラボ　/ポケモンカード" && npm run preview
   ```
   - 表示URLをユーザーに伝え、トップ→地方→都道府県→市→店、フッターの47都道府県、
     オンライン応募ページ、応募条件の表示を目視確認してもらう。

5. **結果を報告**: OK／要修正を明確に伝える。OKなら `poke30-deploy` を促す。

## チェック観点（人の目で見るポイント）
- フッターに47都道府県すべてが出ていて、リンク先が正しい県ページになっているか
- 店カードに「方式・対象商品・応募期間・条件・出典・応募ボタン」が出ているか
- 店が0件の県で「まだありません」の案内が出るか
- 応募ボタンのリンク先（url）が正しいページに飛ぶか
